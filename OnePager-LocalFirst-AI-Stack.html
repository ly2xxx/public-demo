The Most Interesting Project I've Architected: the Journey Itself

Yang Li  ·  a self-funded AI engineering lab, 2023 → today  ·  shipped to PyPI & Docker Hub
"Since 2023 I've rebuilt my own stack every time the frontier moved — not to chase novelty, but because each step solved a real enterprise adoption blocker: trust, orchestration, integration, visibility, and proof. Each chapter below was born from the previous one's pain."
2023

RAG Chatbot

Vanilla RAG with LangChain: local embeddings + FAISS over my own documents; answers grounded in sources, not model weights.

Blocker solved: trust. "LLMs make things up" → ground every answer in your own data.
→
2024

Multi-Agent Flows

LangGraph supervisor pattern — an Internet Researcher orchestrating search, scraping and summarising as a state machine; separation of concerns over one mega-prompt.

Blocker solved: real workflows. One prompt can't run a process → explicit orchestration, checkpointed state.
→
2025

Open Tool Standard (MCP)

Early MCP adopter → built and published md-mcp: a containerised knowledge-base server (PyPI + Docker Hub, helm). Read-only data mounts; any MCP client — Claude Desktop or my own agents — gets the same tools.

Blocker solved: integration. Every data source a bespoke connector → one open standard, secure by design.
→
2026 · Jun

Observability

OpenTelemetry → Collector → Tempo + Prometheus + Grafana, fully self-hosted. Every agent node and LLM call traced — latency, tokens, errors. Privacy-aware spans: call shape recorded, never content.

Blocker solved: visibility. Compliance can't ship traces to vendor clouds → own the telemetry, see every token.
→
2026 · Jul

Evaluation

Gherkin → pytest-bdd → DeepEval: plain-English specs anyone can read, deterministic tool-routing assertions, run as a matrix across a model portfolio — pick the cheapest model that passes, with evidence.

Blocker solved: proof. "Does the agent behave?" answered by tests, not demos — readable by governance, not just engineers.
One connected system today: the LangGraph agent auto-spawns md-mcp in Docker as its knowledge tools (one env var), traced end-to-end, gated by tests. The model itself is a config decision — frontier cloud models via one gateway today, fully local models one line away when data sensitivity demands.

The approach (what held constant across five chapters)

  • Solve the blocker, not the trend: every build answers a specific reason enterprises say "no" to AI — trust, workflows, integration, visibility, proof.
  • Own the seams: open standards at every boundary (MCP for tools, OTLP for telemetry, one gateway for models) so any vendor piece swaps without touching agent code.
  • Secure & responsible by design: read-only mounts, telemetry that records shape not content, sensitivity-based local/cloud routing, answers that cite or refuse.
  • Ship, don't just study: each chapter ends in a public artifact — package, image, dashboard, test suite — not a bookmark.

Evidence on request

  • pip install md-mcp — live on PyPI; Docker Hub image; helm chart.
  • Tempo trace waterfall — per-node latency + token counts; found a 15s scraper bottleneck. You can't tune what you can't see.
  • Grafana — request rate, error ratio, p50/95/99, token throughput per agent; self-hosted, zero external dependencies.
